# Isochrone Map Animation: Design Logic Decoded

This document details how we used Python to transform the street network around the coordinates `41.321254, 2.074613` (El Prat de Llobregat, Barcelona) into a cyberpunk-style "isochrone animation" depicting travel time expansion.

---

## 🗺️ Core Concept: What is an "Isochrone"?
An **Isochrone** represents all points accessible from a specific origin within a given time threshold.
Unlike a simple "circular radius" (e.g., a 3km circle), an isochrone accurately reflects real-world constraints such as street layouts, one-way roads, pedestrian-only zones, rivers, and overpasses. It shows the *actual* reachable area.

## 🛠️ Tech Stack & Tools
This project leverages five core Python data science libraries:
1. **OSMnx:** Responsible for fetching real-world street vector data directly from OpenStreetMap.
2. **NetworkX:** Handles the complex topological graph structure, calculating shortest paths and travel times.
3. **GeoPandas:** Manages geospatial projections and transformations (converting Lat/Lon degrees ↔ actual meters).
4. **Matplotlib:** A powerful plotting tool used to turn raw data into neon-textured visual imagery.
5. **ImageIO:** Stitches the generated static plots together into a fluid GIF animation.

## ⚙️ Step-by-Step Logic

### Step 1: Extracting Earth's Veins (Acquiring the Network)
We feed the program the base coordinates `(41.321254, 2.074613)` and set a maximum range (e.g., the distance covered in a 20-minute drive).
Using `osmnx.graph_from_point()`, the program requests all relevant streets within this bounding box from the cloud.
*   For `walk`, it fetches footpaths, sidewalks, and park trails.
*   For `bike`, it fetches bike lanes and standard roads.
*   For `drive`, it fetches highways, one-way streets, and major arterials.

### Step 2: Converting Distance to Time (Weight Calculation)
Real-world roads have physical lengths (in meters). To convert this to time, we establish average speeds for different transport modes:
*   **Walking Speed:** ~4.5 km/h ➔ **Converted to ~75 meters/minute**
*   **Biking Speed:** ~15.0 km/h ➔ **Converted to ~250 meters/minute**
*   **Driving Speed:** ~36.0 km/h (Urban average) ➔ **Converted to ~600 meters/minute**

The program then iterates over tens of thousands of street edges, dividing each road's "length" by the designated "speed" to calculate the exact minutes required to traverse that specific segment. This value is stored as edge data.

### Step 3: The Ripple Effect (Dijkstra's Algorithm via Ego Graph)
This is the core of the animation! We utilize classic graph theory algorithms (implemented here via `networkx.ego_graph`, which builds upon Dijkstra's algorithm for shortest paths).
We initialize a time counter `t` (e.g., 1 min, 3 mins, 5 mins...).
When `t = 5`, the program starts from the base node and explores connected roads outward. As long as the cumulative travel time is under 5 minutes, it continues.
All intersections (nodes) reached within this 5-minute threshold are marked as "accessible."

### Step 4: Cyberpunk Rendering (Matplotlib Styling)
To achieve a sci-fi aesthetic, we set the map's background color to pure black (`bgcolor='black'`).
The nodes identified in Step 3 as "accessible" are painted with glowing, neon colors:
*   **Walking:** Cyan (`#00ffff`)
*   **Driving:** Neon Pink (`#ff00ff`)
Nodes that are currently inaccessible remain invisible. We then use `matplotlib` to take a "snapshot" of this glowing moment.

### Step 5: Directing the Movie (GIF Stitching)
As the timer `t` progresses from 1 minute up to 30 minutes, we generate a sequence of incremental snapshots (frames).
Finally, we pass these frames to `ImageIO`, which stitches them together, pausing for 0.4 seconds per frame. The result is a vibrant, expanding animation of urban accessibility! 🚀
